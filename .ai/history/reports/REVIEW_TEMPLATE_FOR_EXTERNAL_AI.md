# 実行計画レビュー依頼テンプレート

**目的**: 他のAI（Claude、GPT-4など）に実行計画をレビューしてもらうためのテンプレート

---

## レビュー依頼文（コピーして使用）

```
以下の技術スタック移行実行計画をレビューしてください。

【レビュー観点】
1. 計画の完全性：必要な手順がすべて含まれているか
2. リスク評価：潜在的なリスクが適切に識別されているか
3. ロールバック戦略：各フェーズのロールバック手順が実現可能か
4. 依存関係：フェーズ間の依存関係が正しく整理されているか
5. 検証方法：動作確認の方法が適切で実現可能か
6. タイムライン：各フェーズの期間見積もりが妥当か
7. 技術的正確性：技術的な手順が正しいか（特にSupabase、Prisma、TypeScript関連）

【計画書】
（以下にMIGRATION_EXECUTION_PLAN.mdの内容を貼り付け）
```

## 使用方法

1. **レビュー依頼文**をコピー
2. `MIGRATION_EXECUTION_PLAN.md`の内容を添付
3. 他のAI（Claude、GPT-4など）に送信
4. レビュー結果を**レビュー結果サマリー**に記録
5. 指摘事項を`MIGRATION_EXECUTION_PLAN.md`に反映
6. 必要に応じて複数AIでレビューし、**レビュー比較表**で集約

---

## 注意事項

- レビューAIには、プロジェクトのコードベースへのアクセス権限がないことを前提とする
- 計画書に必要な情報がすべて含まれていることを確認してからレビュー依頼する
- 技術的な詳細が不足している場合は、Cursorで補完してからレビュー依頼する
- 複数AIでレビューする場合は、それぞれ独立してレビューしてもらう（他のAIのレビュー結果を見せない）
