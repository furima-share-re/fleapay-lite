# ドキュメント索引（人間の入口）

**このファイルは人間が最初に読む唯一の入口です。**

## このリポジトリの構造

### 正（Source of Truth）

- **`truth/**`** : 数値・条件・制約の唯一の正
  - 索引: `truth/_index.yml`
- **`spec/**`** : API/DB等の機械仕様の正

### 説明・根拠

- **`docs/**`** : 説明用（正ではない）
- **`adr/**`** : 意思決定の根拠（なぜそうしたか）

## 読み方

1. **まず `truth/**` と `spec/**` を読む**（断言・仕様）
2. **次に `adr/**` を読む**（根拠）
3. **最後に `docs/**` を読む**（背景理解のみ）

## ドキュメント一覧

### truth（正）

- `truth/_index.yml` - truth ファイルの索引・依存・影響範囲

### spec（機械仕様）

- `spec/openapi.yml` - OpenAPI仕様

### docs（説明）

- `docs/01_overview/` - 概要・企画資料
  - `EDO_ICHIBA_F1最小機能セット_広告メディア機能整理.md`
  - `大井競馬場フリマ_インバウンド来場者ペルソナ詳細資料.md`
- `docs/02_spec/` - 仕様説明
  - `API仕様_売主用注文詳細取得.md`
- `docs/03_ops/` - 運営・運用資料
  - `メモ.md`
- `docs/99_archive/` - アーカイブ

### adr（意思決定ログ）

- `adr/AI駆動開発_ドキュメント管理企画_v2.1.1.md` - ドキュメント管理企画

## 注意事項

- `docs/**` は説明用であり **正ではありません**
- `docs/**` に数値・条件・制約を書いてはいけません
- 実装判断には必ず `truth/**` または `spec/**` を参照してください

## ドキュメント投入方法

Cursor チャット経由でドキュメントを投入する際は、以下の3行ヘッダを付ける。

```
[ROLE]=truth | spec | docs | adr
[PATH]=infra/timeout.yml
[SOURCE]=paste
---
（本文）
```

**ルール：**
- 人間が行う判断は **ROLE と PATH の選択のみ**
- AIが自動的に適切な場所に配備します
