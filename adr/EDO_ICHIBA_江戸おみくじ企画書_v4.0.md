## エグゼクティブサマリー

本企画は、訪日観光客(インバウンド)を対象とした国際フリマ「EDO ICHIBA」において、**UGCを成果指標ではなく広告OSを駆動させる原材料として設計**することで、大井競馬場という単一会場・月8回開催という制約下においても、**単年1億リーチを創出**することを可能にする戦略です。

従来の「ガチャ」方式ではなく、文化的親和性とUGC誘発力に優れた**「おみくじ(Fortune Draw)」**を**デジタルゲーム**として採用し、**現地スタッフ0名・システム/AI運営99%**の完全自動運営体制により、最小限のコストで最大の広告価値を実現します。

### v4.0 主要修正点

- **Phase 4を2段階に分離**: Phase 4-A(大井単体・1億リーチ)とPhase 4-B(多会場展開・4億円収益)
- **公式UGC 1本あたりリーチを10,000に修正**: より保守的で現実的な数値設定
- **収益予測の現実化**: 大井単体で年間3,800万円(CPM 200円想定)
- **おみくじ仕様の明確化**: デジタルゲームのみ(物理景品なし)
- **法的リスク評価の追加**: デジタルゲームとしての位置づけ

| 判断に不要な情報は見ない | 詳細データではなく、判断用ダッシュボードのみ確認。 |
| フェーズをまたぐ意思決定だけ行う | フェーズ移行の可否判断、戦略変更の決定のみ。 |
| 属人的判断を排除 | 作業の正確性より「自動化・再現性」を優先。 |

## フェーズ定義(F0〜F4)

各フェーズは**「検証すべき仮説」**を持ち、経営者は**「続行/改善/中止」の判断のみ**を行う。日常作業・データ収集・分析はすべて自動化される。

### 🔹 F0｜決済基盤検証フェーズ(完了)

- **フェーズ目的**: EDO ICHIBA上で「購入」が成立するかを確認する
- **状態**: 2025年11月 α版リリース、決済機能を3店舗で実運用中
- **経営者の役割**: 継続 or 次フェーズ移行の判断のみ

### 🔹 F1｜UGC成立検証フェーズ(現在〜)

- **フェーズ目的**: おみくじ導入により、購入者が実際にSNS投稿を行うかを検証する
- **検証テーマ**:
  - 投稿率は何%か
  - おみくじ有無で行動は変わるか
  - 1投稿あたりの実リーチはどの程度か
- **成功判断ライン**:
  - **投稿率15%以上** → 続行
  - **投稿率20%以上** → 強い
  - **1投稿300リーチ以上** → 拡張検討

### 🔹 F2｜出店者拡張フェーズ

- **フェーズ目的**: 出店者を拡大しても、販売成果・投稿成果が再現されるかを検証する

### 🔹 F3｜メディア完成フェーズ

- **フェーズ目的**: EDO ICHIBAを「イベント」ではなく「UGCメディア」として成立させる
- **状態定義**:
  - 公式UGCが自動生成・配信される
  - 開催日以外もリーチが積み上がる

### 🔹 F4-A｜大井競馬場単体・1億リーチ達成

- **フェーズ目的**: **単一会場・月8回開催で年間1億リーチを安定的に達成する**
- **年間決済数**: 129,600件
- **想定収益**: **年間3,800万円**(決済手数料1,880万円+広告収益1,920万円)

### 🔹 F4-B｜多会場展開・4億円収益達成(将来)

- **フェーズ目的**: **Phase 4-Aの成功モデルを約12会場に展開し、4億円収益を達成する**
- **必要決済数**: 150万件
- **必要会場数**: 約12会場
- **経営者の役割**: 拡張判断・提携交渉・事業化の意思決定のみ

## KPI設計【完全再設計版】

### KPI設計の基本原則(宣言文)

> 人・会社・市場・広告主の判断に依存する指標はKGI/KPIに置かない。  
> それらは「成果目標」「ベンチマーク」「参考指標」として別枠管理する。

**我々が評価するのは、「広告が売れたか」ではなく、  
「広告が自然発生する装置が、止まらず動いたか」だけです。**

## 収益シミュレーション(Phase 4-A)

### 決済手数料収益(確実性: 高)

**計算式:**
- 年間決済数: 129,600件
- 平均決済額: 1,500円
- 決済手数料率: 7% + 固定40円/件
- 1決済あたり収益 = 1,500円 × 7% + 40円 = 145円
- **年間収益 = 129,600件 × 145円 = 18,792,000円 ≈ 1,880万円**

### 広告収益(Phase 4-A想定)

**前提:**
- 公式UGC総リーチ: 9,600万
- CPM(1,000リーチあたり単価): **200円**(保守的)

**広告収益 = 9,600万リーチ ÷ 1,000 × 200円 = 19,200,000円 ≈ 1,920万円**

### Phase 4-A総収益

| 収益源 | 保守的<br>(CPM 150円) | 標準的<br>(CPM 200円) | 楽観的<br>(CPM 300円) |
|-------|---------------------|---------------------|---------------------|
| 決済手数料 | 1,880万円 | 1,880万円 | 1,880万円 |
| 広告収益 | 1,440万円 | 1,920万円 | 2,880万円 |

## おみくじ仕様と法的位置づけ

### おみくじ基本仕様

| 項目 | 内容 |
|-----|------|
| **形式** | **デジタルゲームのみ** |
| 物理的景品 | **なし** |
| 参加条件 | EDO ICHIBAでの決済完了後、QRコードスキャン |
| おみくじ内容 | 吉凶結果(大吉・吉・中吉・小吉・凶 等)、メッセージ、ビジュアル |
| 多言語対応 | 英語・中国語(簡体字/繁体字)・韓国語・日本語 |
| 特典 | なし(ゲーム体験のみ) |

### 法的位置づけ

**デジタルゲームのメリット(景品表示法上)**:
- 物理的な景品がないため、**景品規制の対象外**になる可能性が高い
- 「ゲーム体験の提供」として整理しやすい
- コスト構造がシンプル(物流・在庫管理不要)

**確認すべきポイント**:
- 「おみくじ結果に応じた特典(割引・次回優待等)」がある場合は要注意
- スポンサー露出が「景品」と見なされる可能性
- **Phase 1開始前に、デジタルゲームとしての位置づけを法務確認推奨**
- 想定コスト: 5-15万円

## リスクと対応策

### 技術的リスク(中程度)

| リスク | 対応策 | 優先度 |
|-------|-------|-------|
| AI自動化率99%の実現性 | 例外処理の明確な定義、自動判定不可能なケースは「保留キュー」、週次でまとめて経営者判断 | 高 |
| システム障害 | 冗長化設計、自動復旧機能、緊急度の高い障害のみリアルタイム通知 | 高 |

### 前提条件の連鎖仮定リスク(高リスク)

Phase 4-Aは5つの仮定が連鎖している構造。

**影響分析**: 5つの仮定が各10%減 → 0.9⁵ = 0.59 → 総決済数41%減

**対策: 各フェーズにGo/No-Go判断マトリクスを設定**

| KPI達成 | 成果目標達成 | 判断 |
|---------|------------|------|
| ○ | ○ | 次フェーズ移行 |
| ○ | × | 戦略レビュー後、条件付き移行 |
| × | ○ | KPI設計見直し |
| × | × | 中止または大幅方針転換 |

### 💡 成功の鍵: 9,600本という圧倒的な母数

1本の質に頼らず、**「数」で勝負**する設計。  
公式UGC 1本あたり10,000リーチ(保守的)でも、**9,600本 × 10,000 = 9,600万リーチ**を実現。

### ⚠️ 4億円収益にはPhase 4-B(多会場展開)が必須

**現在**: 大井競馬場単体 → 年間3,800万円  
**目標**: 4億円収益 → 約12会場展開が必要
